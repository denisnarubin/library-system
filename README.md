# Библиотечная информационная система (Library Management System)

Современное веб-приложение для управления библиотекой, разработанное на React + Vite.

## 📋 Описание

Это фронтенд-приложение для управления библиотечными процессами, включая:
- Управление читателями (студенты, преподаватели, разовые читатели)
- Управление книжным фондом
- Выдача и возврат книг
- Учёт штрафов и утерянных книг
- Межбиблиотечный абонемент (МБА)
- Статистика и отчёты

## 🚀 Технологии

- **React 18** - UI библиотека
- **Vite** - Сборщик проектов
- **React Router DOM** - Маршрутизация
- **Axios** - HTTP клиент
- **CSS3** - Стилизация (без фреймворков)

## 🛠️ Установка и запуск

### Предварительные требования
- Node.js 18+ 
- npm или yarn

### Установка зависимостей
```bash
npm install
```

### Запуск приложения
```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:5173

### Сборка для продакшена
```bash
npm run build
```

## 🗄️ База данных

Приложение разработано для работы с PostgreSQL базой данных со следующей структурой:

### Основные таблицы:
1. **reader_categories** - Категории читателей
2. **readers** - Читатели
3. **students_details** - Данные студентов
4. **teachers_details** - Данные преподавателей
5. **one_time_readers_details** - Разовые читатели
6. **publication_points** - Пункты выдачи
7. **library_cards** - Читательские билеты
8. **books** - Книги
9. **authors** - Авторы
10. **book_authors** - Связь книг и авторов
11. **book_copies** - Экземпляры книг
12. **book_loans** - Выдача книг
13. **fines** - Штрафы
14. **lost_books** - Утерянные книги
15. **loan_bans** - Лишение права пользования
16. **interlibrary_orders** - МБА заказы
17. **book_requests** - Статистика заказов

## 📱 Функциональность

### 1. Панель управления (Dashboard)
- Статистика по читателям, книгам, выдачам
- Последние выдачи книг
- Список задолжников
- Быстрые действия

### 2. Управление читателями
- Просмотр всех читателей
- Фильтрация по категориям
- Поиск по ФИО, телефону, email
- Добавление/редактирование/удаление
- Поддержка разных типов читателей

### 3. Управление книгами
- Каталог книг
- Поиск по названию, ISBN, издательству
- Управление экземплярами
- Привязка авторов

### 4. Выдача книг
- Оформление выдачи
- Возврат книг
- Автоматический расчёт сроков
- Отслеживание просрочек

### 5. Статистика и отчёты
- 20 наиболее заказываемых книг
- Задолжники (>10 дней)
- Поступления и утери за год
- МБА заказы по периодам
- И другие отчёты

## 🎨 Особенности дизайна

- **Адаптивный дизайн** - работает на всех устройствах
- **Тёмная боковая панель** - современный вид
- **Иконки-эмодзи** - интуитивная навигация

## 🔧 Настройка

### Изменение API endpoint
Откройте `src/services/api.js` и измените:
```javascript
const API_BASE_URL = 'YOUR_API_URL_HERE';
```

### Добавление новых страниц
1. Создайте компонент в `src/pages/`
2. Добавьте маршрут в `src/App.jsx`
3. Добавьте пункт меню в `src/components/Layout.jsx`

### Изменение стилей
- Глобальные стили: `src/App.css`
- Стили макета: `src/components/Layout.css`
- Стили страниц: `src/pages/Common.css`

## 📝 Заметки по разработке

### Для разработчиков
- Все API вызовы централизованы в `src/services/api.js`
- Компоненты используют функциональный подход с хуками
- Маршрутизация через React Router v6
- Стили написаны на чистом CSS (без препроцессоров)

### Требования к бэкенду
Бэкенд должен предоставлять REST API endpoints для всех сущностей:
- CRUD операции для всех таблиц
- Поиск и фильтрация
- Статистические запросы (функции БД)
